Diseño Técnico — Sistema de Upload de Video Vidalis

Enfoque B: Flutter + Cloudinary directo, backend como orquestador ligero

🏗️ Capas del sistema

Capa 1 — Flutter
  • VideoSource — galería / cámara / URL
  • VideoCompressor — 1080p con video_compress
  • ChunkedUploader — divide y sube a Cloudinary
  • UploadQueue — estado global en Provider
  • UploadBanner — widget flotante persistente
  • LocalNotifier — notificación al terminar
Capa 2 — Backend (Node.js)
  • POST /cloudinary-signature — ya existe
  • POST /videos — registra video en Supabase
  • PATCH /videos/:id — actualiza estado/URL
  • Sin cambios de upload directo
Capa 3 — Cloudinary
  • Recibe chunks de ~6MB
  • Ensambla el video final
  • Genera URL HLS para streaming
  • Thumbnail automático
  • Upload ID para reanudar

🔄 Flujo completo de upload

1
El artista elige la fuente — Galería (image_picker), Cámara (camera package), o pega una URL remota. El sistema crea un objeto UploadJob con estado pending y lo agrega al UploadQueue.
2
Compresión local — VideoCompressor.compress(file, quality: VideoQuality.MediumQuality). Reduce a 1080p. El banner aparece con estado "Preparando video...". Si es URL remota, se omite este paso.
3
Firma de Cloudinary — GET /api/vidalis/cloudinary-signature al backend. Retorna uploadId, signature, timestamp, apiKey. El uploadId se persiste en SharedPreferences para poder reanudar si se cierra la app.
4
Upload en chunks — ChunkedUploader divide el archivo en bloques de 6MB y los sube secuencialmente a https://api.cloudinary.com/v1_1/.../video/upload con el header X-Unique-Upload-Id. El banner actualiza el % real por chunk completado.
5
Registro en backend — El último chunk retorna la secure_url de Cloudinary. Se llama POST /api/vidalis/videos con la URL. El backend guarda en Supabase y dispara el análisis de IA.
6
Notificación + limpieza — El banner muestra "¡Video subido!" con check verde. LocalNotifier dispara una notificación del sistema. Se elimina el archivo temporal comprimido del teléfono. Se limpia el uploadId de SharedPreferences.
↩
Reanudación tras fallo — Si la app se cierra o se pierde señal, al volver se detecta el uploadId guardado en SharedPreferences. Se consulta a Cloudinary cuántos bytes recibió y el ChunkedUploader reanuda desde el siguiente chunk pendiente.

📦 Nuevos paquetes Flutter requeridos

camera: ^0.11

Grabación in-app. Reemplaza image_picker para la fuente cámara.

video_compress: ^3.1

Compresión local a 1080p antes de subir.

flutter_local_notifications: ^17

Notificación del sistema al completar el upload.

connectivity_plus: ^6

Detectar pérdida de red para pausar/reanudar chunks.

⚠️ workmanager no se incluye: Cloudinary chunked upload es suficientemente robusto y workmanager en iOS tiene restricciones severas del OS.
